# We Grow Marathon 🌱

Командный шаговый марафон в Telegram. Концепция «Мы растём, когда двигаемся»:
каждый участник «выращивает» растение, шаги — энергия роста, команда — общий сад.

- **17–23 июля** — «Прорастание»
- **24–30 июля** — «Рост»
- **31 июля – 7 августа** — «Цветение»
- **8 августа** — церемония с итогами и номинациями

Все диалоги ведёт маскот — росток **Гроу 🌱**.

## Из чего состоит система

| Часть | Стек | Роль |
|-------|------|------|
| Бот | Python + aiogram (или Node + grammY) | диалоги, приём скринов, начисление баллов, пуши |
| Backend + БД | PostgreSQL | баллы, серии, команды, согласия, аудит скринов |
| Mini App | React + Telegram Web App SDK | «Мой прогресс», «Моя команда», «Лидерборд» |
| Планировщик | APScheduler / cron | напоминания и регулярные рассылки по расписанию |

Хостинг — один небольшой VPS. Пуши — это обычные `sendMessage` от бота
(отдельный пуш-сервис не нужен), рассылка идёт через очередь с задержкой из-за
лимита ~30 msg/s.

## Быстрый запуск

Нужны Docker (для PostgreSQL) и Python 3.11+.

```bash
# 1. Поднять PostgreSQL (схема применится автоматически)
docker compose up -d db

# 2. Настроить окружение
cp .env.example .env
#   → впишите BOT_TOKEN от @BotFather и свой Telegram ID в ADMIN_IDS

# 3. Установить зависимости
pip install -r requirements.txt

# 4. Запустить бота (схема БД применяется на старте)
python -m bot.main
```

После старта откройте бота в Telegram и отправьте `/start`.

Тесты логики баллов: `cd backend && python3 test_scoring.py`.

Что уже работает (итерация 1): онбординг с согласием и выбором команды по
заполненности, приём шагов (фото сохраняется + ручной ввод числа),
начисление баллов и серий с мгновенной обратной связью, планировщик
(21:00 напоминание, вс сводка, пн лидерборд), админ-команды `/broadcast`,
`/dq <id>`, `/leaderboard`. OCR, mini app и «сад» — итерация 2
(см. [docs/mvp-plan.md](docs/mvp-plan.md)).

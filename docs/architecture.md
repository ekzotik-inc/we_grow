```
                 ┌─────────────────────────────┐
   Telegram  ◄──►│            Бот               │
   (участники,   │  aiogram / grammY            │

                 │  - приём скринов + OCR       │
                 │  - мгновенная обратная связь │
                 │  - пуши (sendMessage)        │
                 │  - /broadcast, админ-лента   │
                 └───────────┬─────────────────┘
                             │ вызовы бизнес-логики
                 ┌───────────▼─────────────────┐
                 │          Backend             │
                 │  scoring.py (баллы, серии)   │
                 │  reconciliation (сводки)     │
                 │  leaderboard (кэш рейтинга)  │
                 └───────────┬─────────────────┘
                             │
                 ┌───────────▼─────────────────┐
                 │        PostgreSQL            │
                 └───────────▲─────────────────┘

   Telegram  ◄──►┌───────────┴─────────────────┐

   └──────────────────────────┘
```

## Ключевые решения

- **Пуши = `sendMessage`.** Отдельный пуш-сервис не нужен. Telegram доставляет
  как нативный пуш всем, кто нажимал `/start`. Массовая рассылка — через очередь
  с задержкой (лимит ~30 msg/s); на 100–200 участников это секунды.
- **Mini App авторизуется через `initData`** — ID пользователя приходит от
  Telegram, отдельных логинов нет. Backend валидирует подпись `initData`.
- **Постоянная кнопка меню** настраивается через `setChatMenuButton` (или
  BotFather) и открывает Web App внутри Telegram.
- **Бизнес-логика — в backend**, бот и планировщик её только вызывают. Правила
  начисления централизованы в `backend/scoring.py`, чтобы бот и проверка
  недельных сводок считали одинаково.
- **OCR — вторая итерация.** На старте: скрин + ручной ввод числа (см.
  docs/mvp-plan.md).

## Потоки данных

1. **Приём шагов:** фото → (OCR или ручной ввод) → подтверждение → `daily_entries`
   + пересчёт `streaks` → мгновенный ответ с баллами и серией.
2. **Недельная сводка:** сверка `reported_total` vs `computed_total` →
   `reconciled` или ручная модерация.
3. **Лидерборд:** пересчёт при начислении, кэш → отдаётся в Mini App и пуши.

# Mini App (Telegram Web App)

Мини-приложение открывается постоянной кнопкой меню бота. Три таба:

- **Мой прогресс** — растение с текущей стадией роста, всего баллов, серия,
  статус за сегодня, календарь марафона (17.07–07.08) с подсветкой по баллам.
- **Моя команда** — место в рейтинге, сумма баллов, состав и вклад участников.
- **Лидерборд** — рейтинг всех команд и топ-10 участников.

## Устройство

Один веб-сервис на FastAPI ([`webapp/server.py`](../webapp/server.py)):

- отдаёт статичный фронтенд [`webapp/static/index.html`](../webapp/static/index.html)
  (self-contained: HTML + CSS + JS, Telegram Web App SDK, тема Telegram);
- эндпоинты `/api/me`, `/api/team`, `/api/leaderboard` — данные из той же базы
  (переиспользует `bot/db.py`).

**Авторизация** — по `initData`: фронтенд шлёт его в заголовке `X-Init-Data`,
сервер проверяет подпись HMAC по `BOT_TOKEN` (Telegram-спецификация) и достаёт
`telegram_id`. Логинов/паролей нет.

**Стадии растения** (по сумме баллов, «Цветение» также при серии 7+):
🌱 Семечко → 🌿 Росток → 🪴 Кустик → 🌳 Дерево → 🌸 Цветение.

## Деплой на Render

`render.yaml` описывает веб-сервис **wegrow-webapp** (free, Oregon). После
Blueprint/ручного создания:

1. Впишите переменные сервиса **wegrow-webapp**: `DATABASE_URL` (тот же Internal
   URL базы), `BOT_TOKEN` (тот же — для проверки initData), `TZ`.
2. Скопируйте адрес сервиса (напр. `https://wegrow-webapp.onrender.com`).
3. В сервисе **wegrow-bot** задайте `WEBAPP_URL` = этот адрес и передеплойте —
   бот выставит постоянную кнопку меню, открывающую мини-апп.

> Free-веб-сервис Render засыпает при простое: первое открытие после паузы
> грузится ~10–30 сек. Апгрейд на платный убирает засыпание.

## Локально

```bash
uvicorn webapp.server:app --reload --port 8000
```
initData подделать нельзя без токена, поэтому API удобнее тестировать через
подписанный initData (см. проверку в истории) или прямо в Telegram.

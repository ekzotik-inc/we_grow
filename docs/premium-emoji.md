# Premium (кастомные) эмодзи в боте

База знаний по механике Telegram Premium Emoji для ботов.
Источник: https://habr.com/en/articles/994622/

## Как это работает

Premium-эмодзи вставляются в текст сообщения тегом `<tg-emoji>` при
`parse_mode="HTML"`:

```html
<tg-emoji emoji-id="5285430309720966085">👍</tg-emoji>
```

- `emoji-id` — ID кастомного эмодзи (из премиум-набора Telegram).
- Внутри тега — **обычный** эмодзи как фолбэк: его увидят те, у кого нет
  Premium, и он же отрисуется, если ID невалиден.

Кнопки: премиум-иконку можно задать полем `icon_custom_emoji_id` у
`KeyboardButton` и `InlineKeyboardButton`.

## Требования и ограничения

- **Владелец бота должен иметь подписку Telegram Premium.** Не сам бот и не
  платный набор стикеров — именно владелец.
- Работает в личных чатах, группах и супергруппах.
- Используются готовые `emoji-id` из существующих наборов (загрузка своих —
  недоступна).
- В aiogram нет отдельного хелпера — используем HTML-тег напрямую.

## Как это включено в проекте

- Модуль [`bot/premium_emoji.py`](../bot/premium_emoji.py): функция `pe(emoji)`
  оборачивает эмодзи в `<tg-emoji>`, если премиум включён и для эмодзи известен
  ID; иначе возвращает обычный эмодзи. Тексты пишутся через `pe(...)`, поэтому
  бот **работает и без Premium** (безопасный фолбэк).
- Бот шлёт сообщения с `parse_mode="HTML"` по умолчанию (см. `bot/main.py`).
- Соответствия эмодзи → ID лежат в [`bot/emoji_ids.json`](../bot/emoji_ids.json).

## Как включить премиум-эмодзи

1. У владельца бота должна быть подписка **Telegram Premium**.
2. Узнать `emoji-id` нужных эмодзи — прямо в боте: админ пишет **`/emojiid`**,
   затем присылает нужные премиум-эмодзи (спорт/бег и т.п.) одним сообщением.
   Бот прочитает их entity и вернёт готовый JSON с гарантированно валидными ID.
   (Альтернатива: getCustomEmojiDocuments в Telegram API.)
3. Впишите пары в `bot/emoji_ids.json` (ключ — обычный эмодзи, значение — ID).
4. Выставьте переменную окружения `PREMIUM_EMOJI=true` (на Render — в
   Environment сервиса).
5. Передеплой. Там, где ID заданы, эмодзи станут премиальными; остальные
   останутся обычными.

> Валидные `emoji-id` нельзя выдумать — неверный ID приводит к ошибке
> отправки. Поэтому ID снимаются с реальных эмодзи через `/emojiid`.

> Пока `PREMIUM_EMOJI` не включён или ID пуст — везде обычные эмодзи, никаких
> ошибок отправки. Так исключён риск «battle-тестить» премиум на живых людях.

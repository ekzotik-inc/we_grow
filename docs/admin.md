# Админ-режим для P&C

Роль `admin` выдаётся по Telegram ID (`participants.role = 'admin'`). В том же
боте открывается служебный функционал.

Панель открывается командой **/admin** (кнопки в главном меню участника нет).

## Проверка результатов (🧾 Проверить результаты)

Шаги проходят модерацию: участник отправляет число + скриншот, результат
попадает в статус `pending` и **не засчитывается**, пока P&C не примет.

- Очередь показывает каждый результат как скриншот + карточку (ФИО, @username,
  команда, дата, заявленные шаги).
- **✅ Принять** — начисляет баллы и пересчитывает серию, участник получает
  уведомление.
- **❌ Отклонить** — не засчитывается; участник может прислать заново.
- **⚠️ Предупреждение** — отправляет участнику комментарий; результат при этом
  НЕ принимается автоматически (остаётся на проверке).

## Управление пользователями (👥 Пользователи)

- Список всех участников с пагинацией и статусом: ✅ подтверждён · ⏳ ждёт ·
  ⛔ дисквалифицирован. Сверху — счётчики.
- **⏳ Заявки на подтверждение** — отдельный список ожидающих с кнопками
  Подтвердить/Отклонить (на случай, если пропустил пуш о регистрации).
- **Карточка участника** (тап по имени): ФИО, @username, ID, команда, статус,
  ASR, дата регистрации и подтверждения, баллы. Действия: подтвердить/отклонить,
  дисквалифицировать/восстановить, перевести в команду, удалить.

## Настройки из панели

- **🖼 Медиа меню** — пришли фото/гиф/видео, и оно станет шапкой главного меню
  у всех (показывается при /start). «-» убирает медиа.
- **🔀 Перевод в команду** — поиск участника по ID/ФИО → выбор новой команды.
- **✏️ Кнопки меню** — переименование кнопок бота (любой текст и обычные эмодзи).
- **🎨 Иконки кнопок** — премиум-иконка кнопки через `icon_custom_emoji_id`
  (Bot API 9.4): пришли премиум-эмодзи, и он станет иконкой кнопки. Нужен
  Telegram Premium у владельца бота и `PREMIUM_EMOJI=true`. (В самом тексте
  кнопки кастомные эмодзи Telegram по-прежнему не рендерит — только как иконку.)
- **📣 Рассылка** — билдер в несколько шагов: текст (**markdown** + премиум-эмодзи)
  → медиа (фото/гиф/видео) → кнопки (своя «текст + ссылка» или «Открыть
  приложение») → предпросмотр → отправка всем.

## Функции

- **Лента входящих скринов** для выборочной проверки: распознанное OCR число
  рядом с фото, расхождения подсвечены (записи с `needs_review = true`).
- **«Запросить проверочный скрин»** (п. 13): бот пишет участнику «P&C просит
  подтвердить сегодняшние шаги в течение часа» и следит за таймером.
- **Дисквалификация в один клик** (п. 14): ставит `disqualified_at`, баллы
  участника автоматически исключаются из командного зачёта, команда получает
  корректное уведомление.
- **Выгрузка в Excel** — все данные (участники, дни, серии, сводки).
- **Ежедневная сводка вовлечённости** — процент сдавших шаги.
- **`/broadcast`** — ручные рассылки (см. docs/notifications.md).

## Что попадает в ленту модерации (`needs_review`)

- ≥ 30 000 шагов (защита от накруток)
- расхождение даты на скрине с датой дня
- расхождение недельной сводки с суммой ежедневных значений
- OCR не смог распознать (для перепроверки ручного ввода)

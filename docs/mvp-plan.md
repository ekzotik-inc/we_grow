# План MVP и приоритеты

Дедлайн старта — **17 июля**, реалистичный срок сборки MVP — 2–3 недели, то есть
впритык. Поэтому режем нестрого-необходимое во вторую итерацию.

## Итерация 1 (к старту 17 июля)

- Онбординг: `/start`, правила + согласие, анкета, выбор команды с заполненностью,
  случайное распределение
- Приём шагов: **скрин + ручной ввод числа** (без OCR), подтверждение, сохранение
  скрина как доказательства
- Начисление баллов и серий (`backend/scoring.py`)
- Мгновенная обратная связь с прогрессом серии
- Планировщик: 21:00 напоминание, вс сводка, пн лидерборд
- Mini App: таб «Мой прогресс» + «Лидерборд» (минимум)
- Админ: лента скринов, дисквалификация, `/broadcast`, выгрузка в Excel

## Итерация 2 (после старта)

- **OCR** распознавания числа и даты на скрине
- Таб «Моя команда» с визуализацией «сада»
- Растение с анимацией стадий роста
- Автоматика «серия под угрозой», «команда поднялась»
- Номинации к церемонии

## Зафиксированные решения по спорным местам

| Вопрос | Решение | Где |
|--------|---------|-----|
| Бонус за серию 7 дней: +4 заменяет или суммируется? | **+4 как максимум, заменяет +2**; начисляется кратно 7 дням | docs/scoring.md |
| Шаги ≥ 30 000 | 3 балла + авто-флаг на проверку P&C | docs/scoring.md |
| Шаги < 5 000 | 0 баллов, но день «сдан» | docs/scoring.md |
| Скрин за чужую дату | OCR сверяет дату → ручная модерация (во 2-й итерации; в 1-й — выборочная проверка P&C) | docs/admin.md |

## Стек и хостинг

- Бот: Python + aiogram (или Node + grammY)
- БД: PostgreSQL
- Mini App: React + Telegram Web App SDK
- Планировщик: APScheduler или cron
- Хостинг: один небольшой VPS

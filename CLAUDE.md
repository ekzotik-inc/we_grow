# Step Together — контекст проекта для Claude

**Читай этот файл первым.** Он описывает актуальное состояние проекта, чтобы
каждая новая сессия сразу понимала контекст.

## Что это

Telegram-бот **Step Together** (@wegrowmarathon_bot) — корпоративный шаговый
марафон для сотрудников компании. Участники ежедневно присылают количество
шагов + скриншот из Fitbit, P&C модерирует, начисляются баллы, команды
соревнуются. Маскот и «ведущий» всех диалогов — росток **Stepy 🪴**
(премиум-эмодзи id `5305729158385533791`); его реплики оборачиваются в
`<blockquote>` через `texts.stepy()`. Админ-сообщения (P&C) — без Stepy,
деловым тоном.

- **Марафон: 17 июля — 7 августа 2026**, таймзона `Asia/Tashkent`.
- Проект **уже запущен в проде, есть живые участники** — миграции только
  аддитивные (`ADD COLUMN IF NOT EXISTS`), данные не ломать.
- Язык всего интерфейса — русский. Обратная связь ведёт на P&C:
  https://t.me/DaryaPMI

## Архитектура (фактическая)

| Часть | Технологии | Где |
|-------|-----------|-----|
| Бот | Python 3.11+, aiogram 3.29, polling, MemoryStorage | `bot/` |
| БД | PostgreSQL (asyncpg), схема `wegrow` (search_path) | `backend/db/schema.sql`, доступ — `bot/db.py` |
| Mini App | FastAPI + uvicorn, фронт — один vanilla-JS файл | `webapp/server.py`, `webapp/static/index.html` |
| Планировщик | APScheduler внутри процесса бота | `bot/scheduler.py` |
| Excel-выгрузка | openpyxl | `bot/export.py` |
| Деплой | Render Blueprint, ветка **`main`** | `render.yaml` |

Каталоги `miniapp/` (React) и `scheduler/` — **устаревшие заготовки первой
итерации, в проде не используются**. Рабочий mini app — `webapp/`.

- Рабочие сервисы: **`wegrow-bot-z6my`** (worker) и **`wegrow-webapp-z6my`**

## Ключевая бизнес-логика

- **Баллы за день** (`backend/scoring.py`): 5000–9999 → 1, 10000–14999 → 2,
  15000–30000 → 3, меньше 5000 → 0. Больше 30000 — на ручную проверку.
- **Серия (стрик)** — дни подряд с 10000+ шагов, считается ТОЛЬКО по
  принятым (`accepted`) дням: `db.recompute_streak`.
- **Недельный бонус** — по календарной неделе пн–вс: 5 дней подряд 10000+
  → +2, 7 дней → +4 (высший, один раз за неделю). Пересчитывается при
  КАЖДОЙ модерации (`db.recompute_weekly_bonus`), не только в воскресенье.
- **Модерация**: заявка `pending` → админ Accept / Reject / Warning
  (warning НЕ принимает результат). Баллы и серия участнику идут только
  после Accept. Повторная сдача за день сбрасывает статус в `pending`.
- **Регистрация**: /start → правила+согласие → телефон (только свой
  контакт) → ФИО → команда (8–10 мест) → заявка админам с Approve/Reject.
  До Approve участник не допускается к сдаче шагов.
- **Главное меню** для подтверждённого участника — персональная карточка
  (`texts.profile_card` + `db.participant_summary`): ФИО, команда с местом,
  шаги, баллы с личным местом, серия.

## Важные соглашения

- **Премиум-эмодзи**: `texts.pe("…")` → `<tg-emoji>`; работают только при
  Telegram Premium у владельца; маппинг — `bot/emoji_ids.json`. Номера
  правил — покифровые keycap (`texts.num`, 11 → 1⃣1⃣).
- **Порядок роутеров** в `bot/main.py`: `onboarding` ПЕРВЫМ (команды должны
  выходить из любых FSM-состояний), затем admin, admin_settings, steps.
- **Скриншоты**: принимаются и фото, и файлы-изображения; file_id документов
  хранится с префиксом `doc:`. В Excel колонка «Фото» — HMAC-подписанная
  ссылка `/shot/{entry_id}/{sig}` (см. `bot/export.py:shot_sig`), файл
  скачивается на сервере webapp — **токен бота в URL не попадает**.
- **Авторизация mini app**: приоритетно Telegram initData (HMAC), fallback —
  вход по номеру телефона (подписанный токен сессии в localStorage).
- **Напоминания в `bot/scheduler.py` включены** и защищены `_marathon_active()`:
  шлются только в даты марафона (17.07–07.08). Задачи: 20:00 серия под угрозой,
  21:00 не сдал шаги, вс 12:00/21:00 недельная сводка, вс 23:55 бонусы,
  пн 10:00 лидерборд.
- **Дедлайн сдачи — 23:55** (`bot/handlers/steps.py:_day_closed`), после —
  `texts.DAY_CLOSED`. **Анти-дубль**: `daily_entries.screenshot_unique_id`,
  при совпадении админ видит предупреждение. **09:00 — дайджест P&C**
  (`pc_daily_digest`), **23:59 — Excel-бэкап админам** (`nightly_backup`).
- **Еженедельный отчёт**: кнопка «📈 Еженедельный отчёт» работает только в
  вс 22:00–23:55 (`steps.py:_weekly_window`); принимается один скриншот
  недельной статистики Fitbit → `weekly_summaries.screenshot_file_id` +
  `reported_at` (повтор заменяет файл, бонусы не трогает). Пуши: вс 12:00
  анонс, 22:00 открытие, 23:15 last-call не сдавшим. В Excel — лист
  «Недельные отчёты» со ссылками `/wshot/{id}/{sig}`.
- **Webapp**: разделы «Правила» и «Помощь» — аккордеоны (`acc()` в
  index.html), первый раздел правил открыт по умолчанию.
- **Отложенные рассылки**: кнопка «⏰ Отложить отправку» в билдере /broadcast,
  список/отмена — /scheduled. Хранятся в `scheduled_broadcasts` (draft jsonb),
  отправляет job `send_due_broadcasts` раз в минуту.
- Телефоны участников видны только P&C. Все упоминания «ASR» заменены на
  «наша дружная компания / наш дружный коллектив».

## Как запустить локально / проверить

```bash
pip install -r requirements.txt
# локальный PostgreSQL, затем:
BOT_TOKEN=... DATABASE_URL=postgresql://... ADMIN_IDS=<id> python -m bot.main
# webapp:
uvicorn webapp.server:app --port 8000
# unit-тесты скоринга:
python -m pytest backend/test_scoring.py -q
```

Схема БД применяется идемпотентно на старте бота. Для сквозной проверки
логики удобно поднять чистую локальную БД и прогнать цепочку: регистрация →
сдача шагов → модерация → пересчёт серии/бонуса → выгрузка Excel.
